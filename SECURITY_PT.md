# ∅ Política de Segurança ZERO

## Segurança por Design

O ZERO é construído sobre a filosofia de **Segurança por Design** e **Arquitetura Zero Trust**. Acreditamos que a segurança não é uma funcionalidade, mas um requisito fundamental para qualquer Sistema Operacional de Agente Pessoal.

### Pilares Fundamentais

1. **Soberania Local-first**: Seus dados nunca saem da sua máquina, a menos que configurado explicitamente.
2. **Menor Privilégio**: Agentes e ferramentas operam com o escopo de acesso mínimo necessário.
3. **Auditabilidade**: Todas as ações agênticas são registradas na Cripta de Auditoria imutável.
4. **Defesa Ativa**: O Zero Sentinel fornece proteção em tempo real contra injeção de prompt e divulgação de dados sensíveis.

## Relatando Vulnerabilidades

Se você acredita ter encontrado uma vulnerabilidade de segurança no ZERO, por favor, reporte-a de forma privada.

- **E-mail**: `security@zero.local` (ou abra um comunicado de segurança privado no GitHub)
- **O que incluir**: Passos para reprodução, avaliação de impacto e um PoC (Prova de Conceito) minimalista, se possível.

## Auditoria de Segurança

Sempre execute `zero security audit --deep` para verificar sua configuração local. Use a flag `--fix` para aplicar automaticamente as recomendações de endurecimento (hardening).
